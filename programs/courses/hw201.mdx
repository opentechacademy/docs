## Course Overview

HW201 provides a deep dive into the Central Processing Unit (CPU). It strips away marketing hype and complex jargon to explain how processors actually schedule work, why core count isn't the only speed metric, how thermal characteristics throttle performance under load, and how to methodically troubleshoot CPU-related issues.

---

## Course Outline

### Module 1: CPU Big Picture
Start with a simple mental model of what the CPU does.
- **1.1 The CPU as a busy workshop**: Use a workshop analogy to understand cores, threads, instructions, and cache.
- **1.2 Quick Check: CPU big picture**: 3-question check on core scheduling, boost clocks, and scoping user workloads.

### Module 2: Cores, Threads, Speed, and Cache
Break CPU performance into smaller, easier ideas.
- **2.1 Workers, threads, pace, and the nearby shelf**: Understand cores, threads, clock speed, and caches without heavy technical jargon.
- **2.2 Practice: Explain CPU speed to a beginner**: Autograded assignment testing your ability to explain threads, cache hits, multi-threaded workloads, and IPC performance factors.

### Module 3: Heat, Power, and Boost
Learn why CPUs slow down when the support system is not ready.
- **3.1 Why fast CPUs still need support**: Connect CPU performance to cooler selection, thermal paste, motherboard power delivery, and voltage regulation.
- **3.2 Quick Check: Heat and boost**: 3-question check on thermal throttling, temporary boost durations, and diagnostic steps for hot processors.

### Module 4: Choosing a CPU
Choose from the user need instead of from marketing alone.
- **4.1 Match the CPU to the job**: Use real-world user stories to evaluate CPU performance, integrated graphics requirements, and socket compatibility.
- **4.2 Practice: Pick the safer CPU direction**: Autograded assignment on choosing processors for specific home/office use cases and motherboard socket matches.

### Module 5: CPU Troubleshooting
Use symptoms to decide whether the CPU is really the problem.
- **5.1 Follow the waiting line**: Troubleshoot CPU complaints by looking for system bottlenecks (RAM pressure, disk usage, GPU load, etc.).
- **5.2 Quick Check: CPU troubleshooting**: 3-question check on identifying bottlenecks, measuring CPU utilization, and resolving slow application performance.

### Course Review & Final Exam
Review CPU ideas and complete the final quiz.
- **6.1 Course review: What to remember**: Core concepts review summarizing architecture, caches, and troubleshooting.
- **6.2 Final Quiz: CPU Deep Dive**: 6-question cumulative quiz covering thread scheduling, cache routing, cooling impacts, selection constraints, and diagnostic habits.
